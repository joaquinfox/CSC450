C and C++ are part of a group of programming languages that use pointers as aliases for values stored in memory. A pointer stores the location in memory of a value, and we can use it in place of that value. This provides the advantage of not having to duplicate values in memory every time we want to do something with them. Pointers are a variable type that stores addresses, and nothing else. These addresses are physical locations on a memory card and are described by a 12-digit, hexadecimal number (addresses are also referred to as locations or references ). We can think of pointers as maps to a physical location. Instead of an actual map though, the address will be the hexadecimal number that encodes a location.

Since pointers only store location addresses, the following is not allowed: string *foo = "save me!!!" // Not allowed. Instead, we assign pointers to a reference, which is a location in memory, that holds some value of interest to us. We can do this like so:


        string foo = "I'm over here!!;
        string *pointMeToFoo = &foo;

We can now pass the pointer around as an argument and it will always and forever point to the location of the variable foo. This will be true even after the value of foo is changed or destroyed, in which case the pointer will point to a null location.

Pointers in C++ have two available operations: they can point to a location (reference a location) and they can grab the value at that location (de-reference the location).

Among the benefits of using pointers are more efficient code, faster execution, and memory savings (Backman, Kjell). We can pass pointers to objects, instead of copies of objects, which reduces both space and time complexity. We can immediately delete those values when we are done with them, and this helps make the code fast. Even today you hear developers talk about how fast the C languages are. In part, this is due to the ability to manage memory efficiently, through pointers. It's also due to the relatively low level of the language, and its relatively small size (see Quora reference below).
