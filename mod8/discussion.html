Guideline 42 of Java Coding Guidlines is "Use exceptions only for exceptional conditions.". Truly "exceptional" events are most often going to be known, named and checked events. In general exception handling should be confined to this group and not used for runtime errors in logic.

Exception handling allows us to handle bad events "gracefully". This means that we give the program an opportunity to tidy up, protect important things, put on a seat bealt or a parachute, and send a message to our family, before the actual train wreck. Or, more accurately, we provide a mechanism to save data, close processes, and log a helpful message before the exception causes our program to exit.

The issue, according to Long, et al, is the "nonlocality" of a catch statement. Because it is possible to end up catching an exception far from the source, this can make it harder to trace the exception to its source, although I would think a good error message could help with this. It can also be hard to re-factor (or improve) code written in try- catch blocks for fear of breaking things.

Best practices are to use exception handling methods for checked exceptions. Here is a comprehensive list of checked exceptions in Java. Another best practice is to use the most specific exception we can, and to catch that one first in a catch net.

The question comes up for me, is it ever a good idea to write a custom exception? I don't have an answer for this and would like to know what other people think. When I first learned about custom exceptions I thought they were a super power I could use to protect my code from practically any unforseen event. Now I am not so sure.

A good use (and in fact a required use ) of exceptions in Java is with I/O operations. Java requires that we explicityly write a throws statement for any method that could throw a checked exception (Carrano, Frank & Hernry, Timothy. 2018). Here is an example from an earlier class.


    import java.io.File;
import java.io.FileNotFoundException;
import java.io.FileReader;

class ExceptionHandlingDemo {

    public static void main(String args[]) {

        try {

            // Following file does not exist
            File file = new File("E://file.txt");

            FileReader fr = new FileReader(file);
        } catch (FileNotFoundException e) {
            System.out.println("File does not exist");
        }
    }

}
