This question sent me researching and I think I have an answer.

When a pointer is left pointing to a non-existent value, it becomes a dangling pointer. This is a pointer that points to nothing, and it can grab onto random areas of memory, including areas where the OS is written. If that same pointer is then written to, we can overwrite whatever was there, including pieces of our OS.

Dangling pointers come about under several circumstances, such as declaring a pointer, and initializing it in different scopes, in C ...


    int *ptr;
    {
        int foo = 85;
        ptr = &foo;
    }

It can also come about when the object a pointer pointed to is destroyed.


    char *ptr1, *ptr2;
    char demo1 = '$';
    ptr1 = &demo1;
    ptr2 = ptr1;

    delete ptr1;
